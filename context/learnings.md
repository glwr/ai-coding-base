# Learnings & Gotchas

> Things that surprised us, debugging insights, pitfalls to avoid.

## How to Read
- Entries are in reverse chronological order (newest first)
- Search by feature ID or keyword
- High-priority learnings are marked with **[IMPORTANT]** in the title

## Entry Format
```
### [Learning Title]
- **Date:** YYYY-MM-DD
- **Feature:** PROJ-X (or "General")
- **Skill:** /skillname
- **Learning:** What happened and how it was resolved
- **Rationale:** Why future developers should know this
```
